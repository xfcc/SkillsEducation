# SkillsEducation：专业教育通用架构 (PoC)

> 泛知识类产品兜售“信息焦虑”，而专业教育产品交付“核心胜任力”。

**SkillsEducation** 是一个概念验证小程序项目，旨在探索并确立「专业教育（Professional Education）」品类的产品架构范式。  
以高门槛的心脏电生理（EP）医生培训为业务沙盘，将庞杂的医疗资讯、学术课程与硬核的手术模拟工具，重构为一套高信噪比、SaaS 化的学习工具。这套底层架构同样适用于工程、高端制造、法律、投资等一切依赖「刻意练习」的专业领域。

## 核心设计哲学：知与行的物理隔离

市面上绝大多数教育 App 是「内容大卖场」，用海量信息占据用户时间，却难以兑现「从知道到做到」的跨越。

SkillsEducation 采用一种更冷酷的划分方式：**按信息流向，将「知」与「行」在架构上彻底隔离。**

| 维度 | 理论 (知) | 演练 (行) |
|------|------------|------------|
| **信息流向** | 系统 → 用户（Input） | 用户 → 系统（Output） |
| **目标** | 构建心智模型、建立知识结构 | 暴露技能盲区、形成肌肉记忆 |
| **典型形态** | 课程、病案、文献解读 | 训练营任务、工具实操、自测与闯关 |

在这一原则下，产品不做「内容堆砌」，而是明确：**理论页只负责输入，演练页只负责输出。** 用户先在「知」里建立标准答案，再在「行」里用决策与操作检验自己。

---

## 功能架构：基于知与行的四页结构

产品用四个主 Tab 承载上述分工，并让「枢纽」承担发现与入口、「我的」承担资产与设置。

- **枢纽（首页）**  
  视野与效率入口：搜索、核心训练营轮播、临床前沿信息流、直播浮层。不承担深度学习，只做「看到」与「进入」。

- **理论库**  
  纯输入侧：学术课程（400+ 节）、病例学习（600+ 例）、文献解读（100+ 篇）。全部是系统给出答案、用户吸收的结构化内容。

- **演练**  
  纯输出侧：iEP 训练营与我的班级（结构化任务与研讨）、实用工具（室早定位、AIFV 报告、PVI 诊断、AI 虚拟模拟器等）、每日一题。强调用户做选择、做操作、做自测。

- **我的**  
  学习资产与账号管理：学习时长与积分、专职勋章、专属报告档案、个人资料与设置等。不参与知/行的内容划分，只做汇总与入口。

---

## 项目意义

SkillsEducation 不追求直接上架为完整商业产品，而是验证一件事：**通过「知 / 行」的物理隔离与清晰的功能边界，可以把庞杂的专业资源收敛成一条可理解、可扩展的专业进阶路径。**  
若你所在领域同样依赖「先建立心智模型，再通过刻意练习巩固」，本项目的架构与四页划分可作参考与二次演绎。
