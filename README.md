# SkillsEducation

## 1. 品类定义：

本项目是一个针对「专业教育」品类的范式概念模型，旨在探讨并验证：在复杂的专业领域，产品架构究竟该如何设计，才能跨越“知识传递”，真正实现“技能交付”。

泛知识付费和学科学习提供的是“信息消除未知”与“情绪反馈”，而「专业教育」（如医疗实操、飞行驾驶、复杂工程代码等）交付的则是“核心胜任力（Competence & Skill）”。

* **知识教育：** 解决“是什么”的问题。输入是单向的，检验标准是“记忆与理解”。
* **专业教育：** 解决“在真实/复杂情境下如何做”的问题。检验标准是“情境下的决策与执行能力”。

根据**米勒能力金字塔（Miller's Pyramid）**，大多数数字教育产品停留在底部的 `Knows`（知道）和 `Knows how`（知道怎么做），而我们的目标是利用数字化手段，无限逼近顶层的 `Shows how`（在模拟情境中展示决策）乃至 `Does`（真实操作）。

## 2. 解决方案：

要完成从“传递知识”到“训练技能”的跃迁，跨越米勒金字塔的鸿沟，不能只依赖学习者的自觉。基于 4C/ID（复杂技能教学设计）模型[1]，我们将系统的学习活动按“信息的流向”与“交互的深度”，切分为两个功能模块。这不是简单的 UI 划分，而是对大脑认知机制的物理适配：

**模块一：知（理论 Input）—— 极低阻力的外挂大脑**
结构化的知识库。采用“手册化”折叠与强检索结构。系统给答案，用户负责提取。这里不需要沉浸式的体验，只提供一个随取随用的知识底座，帮用户建立严密、无干扰的知识框架。这里绝对安静。
> 认知负荷理论[2]：面对复杂专业技能时，人类的“工作记忆”极易超载。静态知识的呈现必须极度降噪，才能把极其有限的脑力留给实操环节。

**模块二：行（演练 Output）—— 零延迟反馈的情境沙盘**
情境模拟工具。系统不给教条，用户在各类模拟工具中做决策、下指令。通过人为制造的情境，让用户主动调用知识，同时，系统和算法给予客观的即时反馈。
> 情境认知与刻意练习[3][4]：技能无法在真空中传递，必须在带有摩擦力的真实情境中长出。只有在刺痛的客观反馈中，胜任力的幻觉才会被打破，神经与肌肉记忆才得以重塑。

**学习活动流程：不断成长的技能飞轮**
两个功能模块通过底层的“学分与能力图谱”串联成成长飞轮：用户在「知」获取心智图纸 -> 进入「行」应对情境任务 -> 犯错并暴露盲区后，带着认知缺口返回「知」查漏补缺 -> 最终用「行」的成果解锁更深层的实操权限。

这就完成了从“被动喂养”到“主动索求”的动机转换，构成了「专业教育」产品应有的 SaaS 化骨架。

## 3. 电生理业务上的示例

我们以**心脏电生理（EP）医生培训**为示例业务，把上述结构落进一个小程序：学术与病例资源很多，同时又有室早定位、消融模拟等需要「动手」的场景，适合做知/行分离的演示。

下图是当前小程序在电生理场景下的四页结构示意：

![电生理示例：四页结构](./example/skillsEducation.PNG)

- **首页**：搜索、训练营/课程轮播、临床前沿信息流、直播入口。只做发现和跳转，不承载深度学习。
- **理论**：学术课程、病例学习、文献解读。全部是「系统给答案」的输入型内容。
- **演练**：iEP 训练营与班级任务、室早定位/AIFV 报告/PVI 等工具、每日一题。全部是「用户做选择、做操作」的输出型场景。
- **我的**：学习时长与积分、勋章、报告档案、设置。与知/行无关，只做资产与账号管理。

在电生理这个业务上，知与行的划分直接对应「先学标测与术式、再在训练营和工具里练」的路径；同一套划分方式也可以套到其他依赖「先建立心智模型、再刻意练习」的专业领域。

## 4. 内容与配置：由 CMS 驱动

首页轮播、临床前沿、理论库入口、演练工具等均由**后台 CMS** 配置下发。运营在后台维护栏目、链接、排序与上下架，小程序按配置渲染，便于多业务线复用同一套架构。
