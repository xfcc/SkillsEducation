# SkillsEducation

## 1. 项目意义与行业洞察：

本项目是一个针对「专业教育」品类的范式概念小程序：不面向最终用户交付完整产品，而是探讨「专业教育」这一品类在产品架构上究竟该怎么拆、怎么搭，并做出一套可复用的验证模型。

泛知识付费兜售的是“信息”与“情绪安慰”，而专业教育交付的必须是“核心胜任力（Skill）”。学习一门硬核技能与获取一段纯知识有着本质的物理区别：

* **纯知识（Knowledge）** 是单向输入，解决的是“消除未知”，检验标准是记忆。
* **技能（Skill）** 是神经与肌肉的双向重塑，解决的是“行为转化”，检验标准是在高压/复杂环境下的准确决策与操作。

市面上绝大多数的教育 App 是把传统课堂搬到了线上的“内容大卖场”。它们试图用内容让用户产生“看懂即掌握”的幻觉，却无法支撑“练会”这一最艰难的闭环。

本项目意在验证：**用极度克制的架构将“知识获取”与“技能试错”进行隔离，让专业教育类产品真正成为用户的“刻意练习工作台”。**

---

## 2. 解决方案：

要完成从“传递知识”到“训练技能”的跃迁，产品架构必须按**信息的流向与交互的深度**，将系统划分为两个绝对独立的心智空间：

- **知（理论 Input）：结构化的真理库。** 系统给答案，用户负责吸收。课程、经典病案、文献指南均归于此类。废除碎片化的瀑布流，采用“手册化”折叠，帮用户建立极其严密的底层心智模型。这里是安静的“数字图书馆”。

- **行（演练 Output）：高保真的试错沙盘。** 用户做决策、下指令，系统基于算法给予冷酷的客观反馈。训练营的每日通关任务、AIFV 诊断工具、虚拟模拟器实操均归于此类。目标是打破用户的能力幻觉，暴露盲区，形成记忆。这里是真刀真枪的“靶场”。

通过底层的学分与勋章系统，我们将“知”与“行”串联成飞轮：在图书馆获取图纸，去靶场打出环数，再用环数解锁更深层的机密图纸。这就是“专业教育”产品应有的 SaaS 化骨架。

## 3. 电生理业务上的示例

我们以**心脏电生理（EP）医生培训**为示例业务，把上述结构落进一个小程序：学术与病例资源很多，同时又有室早定位、消融模拟等需要「动手」的场景，适合做知/行分离的演示。

下图是当前小程序在电生理场景下的四页结构示意：

![电生理示例：四页结构](./example/skillsEducation.PNG)

- **首页**：搜索、训练营/课程轮播、临床前沿信息流、直播入口。只做发现和跳转，不承载深度学习。
- **理论库**：学术课程、病例学习、文献解读。全部是「系统给答案」的输入型内容。
- **演练**：iEP 训练营与班级任务、室早定位/AIFV 报告/PVI 等工具、每日一题。全部是「用户做选择、做操作」的输出型场景。
- **我的**：学习时长与积分、勋章、报告档案、设置。与知/行无关，只做资产与账号管理。

在电生理这个业务上，知与行的划分直接对应「先学标测与术式、再在训练营和工具里练」的路径；同一套划分方式也可以套到其他依赖「先建立心智模型、再刻意练习」的专业领域。

---

## 4. 内容与配置：由 CMS 驱动

首页轮播、临床前沿、理论库入口、演练工具等均由**后台 CMS** 配置下发。运营在后台维护栏目、链接、排序与上下架，小程序按配置渲染，便于多业务线复用同一套架构。
